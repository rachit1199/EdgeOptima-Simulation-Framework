1. Problem Statement

The proliferation of smart devices, IoT sensors, and data-driven applications has led to an exponential increase in computational demand at the network edge. Traditional cloud computing architectures, though powerful, suffer from high latency and energy consumption due to their centralised nature. For real-time applications — autonomous vehicles, industrial IoT, augmented reality — round-trip latency to a remote data centre is simply unacceptable.

At the same time, executing all tasks locally on constrained devices drains batteries rapidly. Neither extreme — full local execution nor full cloud offloading — is optimal. What is required is an intelligent, dynamic decision for every incoming task: should this task execute locally, be offloaded to a nearby edge server, or be sent to the cloud?

"The core challenge is the energy–latency trade-off: offloading reduces local energy consumption but incurs transmission cost and delay. The optimal decision depends on task characteristics, network conditions, server load, and application deadlines — simultaneously."

This simulator models exactly this decision problem and allows researchers and engineers to explore how different algorithm choices and infrastructure configurations affect total system energy and QoS compliance.

2. Three-Tier Cooperative Architecture

This project extends the cooperative offloading model proposed by Long et al. (2018) by incorporating dynamic decision-making and adaptive resource allocation across three computational tiers.

01
Local Device Execution
Tasks are executed directly on the IoT device or mobile terminal. No transmission cost, but limited CPU capacity leads to high execution energy for complex tasks. Suitable for lightweight, privacy-sensitive workloads.
Tier 1 — Device
02
Edge Server Offloading
Tasks are offloaded to a nearby Mobile Edge Computing (MEC) server, typically co-located with a cellular base station. Offers low transmission latency and high compute capacity. Optimal for delay-sensitive, mid-complexity tasks.
Tier 2 — Edge
03
Cloud Server Offloading
Tasks are routed through the edge to a remote cloud data centre. Virtually unlimited compute capacity, but higher end-to-end latency due to two transmission hops. Best suited for computationally intensive, deadline-tolerant workloads.
Tier 3 — Cloud

Each task is independently assigned to a tier based on its data size, computational complexity, and latency deadline. The assignment algorithm evaluates the weighted objective function across all three tiers and selects the assignment that minimises total cost.

3. Objective Function & Energy Model

The optimisation objective balances total system energy and end-to-end task latency, subject to per-task deadline constraints. The objective function is defined as follows:

minimise F(x) = we · Etotal + wl · Ttotal
subject to: Ti ≤ Di  ∀ i ∈ Tasks   |   we + wl = 1
where: Etotal = Σ ( Etx,i + Eexec,i )  across all assigned tiers
Etx,i = Ptx · ttx,i   |   Eexec,i = Pexec,tier · texec,i

Here, we and wl are user-configurable weights controlling the trade-off between energy efficiency and latency minimisation. The default configuration is we = 0.60 and wl = 0.40, reflecting a bias toward energy efficiency.

Transmission time is derived from task data size and the available bandwidth on each hop (Local→Edge and Edge→Cloud), while execution time is computed from task complexity (MFLOPS) divided by server processing capacity.

4. Optimisation Algorithms

The simulator implements six offloading strategies, ranging from simple baselines to heuristic and learning-based approaches, allowing direct comparative analysis.

Algorithm Category Description Complexity
Greedy Heuristic Assigns each task to the tier with the lowest immediate objective cost. Fast, deterministic, but may miss global optima. O(n)
Simulated Annealing Meta-heuristic Probabilistic search that accepts worse solutions at high temperature to escape local optima. Converges to near-optimal solutions over iterations. O(n · iter)
Deep RL (DQN) Learning-based A Deep Q-Network agent trained on task profiles. Refines greedy decisions via policy learning, achieving the best average performance across heterogeneous workloads. O(n)
Round Robin Baseline Cycles tasks sequentially across the three tiers regardless of task characteristics. Used as a load-balancing baseline. O(n)
Local Only Baseline All tasks execute on the local device. Represents the worst-case energy scenario for complex tasks, but zero transmission overhead. O(1)
Cloud Only Baseline All tasks are offloaded to the cloud. Minimises device energy but maximises latency due to two-hop transmission. O(1)

5. Alignment with UN Sustainable Development Goals

This research directly contributes to two of the United Nations Sustainable Development Goals by promoting energy-aware, intelligent computing infrastructure at the network edge.

7
Affordable and Clean Energy
By minimising redundant computation and optimising task placement, the framework reduces total energy consumption across distributed IoT ecosystems. Intelligent offloading directly lowers the carbon footprint of edge computing infrastructure, contributing to cleaner, more sustainable digital systems.
9
Industry, Innovation & Infrastructure
EdgeOptima promotes next-generation resilient digital infrastructure through adaptive edge–cloud collaboration. It enables intelligent, scalable computing at the network edge for modern industrial, civic, and consumer applications — fostering innovation in distributed systems design.
